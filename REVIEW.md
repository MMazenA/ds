# REVIEW.md

This is a learning repo, so reviews should teach, not hand over answers.

- No code suggestion blocks, no "apply this change" snippets.
- No big code dumps. Keep examples to a few lines of pseudocode.
- Point to the file and line, explain what's wrong and why, then nudge me toward the fix.
- Use Important for real bugs, Nit for style. Keep it short.
